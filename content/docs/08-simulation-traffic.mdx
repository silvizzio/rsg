The Traffic simulation lets the Operations Manager rehearse the same pressure from the fleet side: how vehicles, chargers, and the road network hold up when demand rises. The shift it gives the operator: from finding out something is wrong too late, to always knowing what is running, what is at risk, and what is out of service. In the Operations Manager view, scoped to Turtle Bay here, Simulation sits alongside Fleet, Utilization, Drivers, and EV.

### Creating an event

Under Event to simulate, New Event opens a builder for a condition the presets do not cover. You name the event and set its effect, then it joins the event list and can be selected into any run alongside the built-in events. This lets the operations persona model a scenario specific to the day rather than only the standard ones.

### Creating an action plan

Under Action plan, View All lists the saved plans and New Plan creates one. A plan is a named set of actions, with the action count shown beside it. Once saved, a custom plan is selectable and forecasts against any event the same way as the built-in plans, so the operations persona can prepare a tailored response and test it before committing.

## Reading the fleet forecast

Read the verdict first, then the metric that drove it: utilization and charging set capacity risk, fatigue and maintenance set continuity risk.

| Metric | What it measures | Watch when |
| --- | --- | --- |
| Utilization at peak | Share of the fleet in use at peak | Approaches 100%, here 96% |
| Idle time | Share of the fleet sitting unused | Very low means over-stretched, very high means waste |
| Charging demand | Load on the EV chargers | Chargers hit capacity, here 3 maxed at 84% |
| Energy needed | Extra energy the scenario draws | Rises against typical, here +620 kWh |
| Maintenance backlog | Vehicles waiting on service | Grows across the shift, here +2 by end of shift |
| Fatigue risk | Drivers near a fatigue limit | Any count above zero, here 3 on the bridge |

<Callout type="warning">
While simulation is on, the alert tray shows scenario alerts, not live ones. Live incidents are held until you close simulation.
</Callout>

## Implementing a plan

Once a run is forecasted, the right panel turns the result into action. It lists the plans the operations persona prepared for the event, each with its forecasted effect and an Implement control. Selecting Implement enacts that plan's actions: the dispatch goes to the fleet and building teams, the action moves into the live operations view, and the forecast baseline re-anchors to track the plan as it runs. This is the step that takes a run from a what-if to an executed response.

A plan can be implemented as forecasted, or adjusted first by reopening it under Action plan, editing its actions, and re-running to confirm the new effect before dispatch.
