Alerts surface in the persona-scoped alert tray and resolve to a site on the map. Every alert moves through a fixed lifecycle, and operators advance it as they respond.

## Alert reference

| Field | Example | Context |
|---|---|---|
| Alert Type | Crowd density | Category of the incident |
| Severity | High | See sub-table below for all values |
| Status | Open | See sub-table below for all values |
| Location | Shura Marina | Site the alert resolves to on the map |

**Severity values**

| Value | Meaning |
|---|---|
| Low | Informational, no action required |
| Medium | Monitor, action may be needed |
| High | Immediate attention required |

**Status values**

| Value | Meaning |
|---|---|
| Open | New incident, not yet acknowledged |
| Acknowledged | Operator has accepted the alert |
| In Progress | Response dispatched |
| Resolved | Closed, logged in history |

## Managing an incident

1. Select an alert in the tray to focus its site on the map.
2. Acknowledge the alert to move it from Open to Acknowledged.
3. Dispatch a response and confirm it advances to In Progress.
4. Mark it Resolved once handled, which logs it to history.

<Callout type="warning">
A High severity alert raised for OPS can be configured to mirror into the GXM tray. Confirm the mirror thresholds against the IOC configuration before relying on them.
</Callout>
