Simulation mode replays a scripted scenario through the twin so operators can rehearse crowd, traffic, and incident response without live data. The map, panels, and alert tray respond to the scenario as if it were live.

## Simulation reference

| Field | Example | Context |
|---|---|---|
| Scenario | Peak arrival | The scripted dataset being replayed |
| Playback | Playing | See sub-table below for all values |
| Speed | 2x | Playback rate relative to real time |
| Timeline | 00:14 / 00:45 | Current position in the scenario |

**Playback values**

| Value | Meaning |
|---|---|
| Playing | Scenario advancing |
| Paused | Held at the current timeline position |
| Stopped | Reset to the scenario start |

## Running a simulation

1. Toggle simulation mode on from the interface or with the simulation shortcut.
2. Select a scenario to load its dataset.
3. Set the playback speed.
4. Play, pause, or scrub the timeline to study the scenario.
5. Toggle simulation mode off to return to live data.

<Callout type="warning">
While simulation mode is on, the alert tray shows scenario alerts, not live alerts. Live incidents are held until you exit simulation mode.
</Callout>
